# Personal Steering Migration Guides

This directory contains migration guides for personal workflow pattern transitions.

## Purpose

When personal preferences, tool usage patterns, or methodologies change, document the migration here. This provides:

- Historical context for why changes were made
- Step-by-step transition instructions
- Timeline and status tracking
- Examples of old and new patterns

## When to Create a Migration Guide

Create a guide here when:

- A tool usage pattern changes (e.g., switching from one MCP to another)
- A methodology evolves (e.g., research protocol refinement)
- Workflow preferences shift (e.g., memory tagging strategy changes)
- Cross-cutting behavioral patterns change (e.g., IP protection rules)

For minor refinements, update the steering file in place. Migrations are for significant pattern changes.

## Naming Convention

Use format: `YYYY-MM-description.md`

Examples:

- `2026-03-personal-steering-lifecycle.md`
- `2026-04-memory-tagging-strategy.md`
- `2026-05-research-protocol-refinement.md`

## Lifecycle

1. **Active**: Migration guide created, transition in progress
2. **Complete**: Pattern fully adopted, old pattern no longer used
3. **Archived**: After 3+ months, move to `_archive/`
