# Sequential Thinking

Use the `sequential-thinking` MCP for tasks involving multi-step
reasoning, debugging, architectural decisions, or structured analysis.

## Always Use Sequential Thinking For

- Debugging complex issues
- Planning multi-file changes
- Evaluating tradeoffs between approaches
- Any task with more than 2 logical steps
- Formulating follow-up research questions based on findings
- Deciding whether to promote a memory to a steering file
- Sanitizing queries before sending to external tools
- Resolving conflicting code patterns using provenance data

## Do NOT Use For

- Simple syntax questions (Tier 1)
- Recalling a stored fact from memory (Tier 2)
- Single-step tasks with obvious answers
