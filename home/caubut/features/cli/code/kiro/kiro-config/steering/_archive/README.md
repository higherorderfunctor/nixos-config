# Archived Personal Steering Files

This directory contains steering files for personal patterns that are no longer in use. These are preserved for historical reference and understanding workflow evolution.

## Archive Index

_No archived patterns yet. This will be populated as personal patterns are deprecated._

## When to Archive

A personal pattern should be archived when:

1. The pattern is no longer used in any workspace
2. Migration to new pattern is complete (check migration guide status)
3. No active references exist in current steering files
4. Minimum 3 months have passed since deprecation notice

## Archive Format

Each archived file should include metadata at the top:

```markdown
---
archived: YYYY-MM-DD
reason: "Brief explanation"
replacement: "path/to/new/pattern.md"
last_used: YYYY-MM-DD
verification: "How pattern obsolescence was verified"
---
```

## Accessing Archived Content

Archived files are searchable via git and can be referenced when needed for:

- Understanding historical workflow decisions
- Comparing old vs new approaches
- Audit and learning purposes
- Personal knowledge preservation

## Long-Term Storage

After 1-2 years, archived patterns may be compressed or removed if they no longer provide value. Keep only what's useful for understanding workflow evolution.
